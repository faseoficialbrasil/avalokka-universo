📜 Crônicas da Invocação

Nem todo véu se abre sem resistência. Cada erro enfrentado foi uma chave, cada solução, um ritual. Aqui estão os registros do caminho percorrido para manifestar Avalokka.

1. Erro 404 no site

🔧 Solução: Colocar index.html na pasta /docs e configurar GitHub Pages corretamente.

2. Cache impedindo visualização

🔧 Solução: Limpar cache ou usar modo anônimo.

3. Caminhos incorretos para arquivos

🔧 Solução: Corrigir os caminhos relativos no HTML.

4. GitHub Pages não atualizando

🔧 Solução: Aguardar propagação ou reconfigurar origem.

5. Nome duplicado de arquivo

🔧 Solução: Renomear corretamente para index.html.

6. CSS não aplicado

🔧 Solução: Garantir que style.css esteja na raiz e com link correto.

7. Áudio não tocando

🔧 Solução: Corrigir caminho e usar <audio> com autoplay.

8. Botão de som sem estilo

🔧 Solução: Adicionar bloco CSS para .controle-audio button.

9. Névoa sem transparência

🔧 Solução: Usar opacity e pointer-events: none.

10. Site funciona só no modo anônimo

🔧 Solução: Limpar cache ou forçar recarregamento.

11. Imagem pesada para celular

🔧 Solução: Usar JPG otimizado e CSS responsivo.

12. Visualização quebrada em celular

🔧 Solução: Adicionar meta viewport e ajustar layout.

13. GitHub Pages apontando para branch errada

🔧 Solução: Configurar origem como main e pasta /docs.

14. Propagação lenta confundida com erro

🔧 Solução: Aguardar 5–10 minutos após commit.

15. Névoa estática

🔧 Solução: Criar animação @keyframes flutuarNevoa e aplicar no body::before.
